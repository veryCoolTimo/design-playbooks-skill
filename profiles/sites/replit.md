## Overview
A dark navy-black tech canvas (#0E1525) alternating with clean white pages, unified by one loud signal color: Replit orange on every "Start building" CTA. Headlines are quiet, tightly-set grotesque type — except signature taglines where individual letters dissolve into pixel blocks, a nod to code and terminals. Big rounded product screenshots and soft pastel gradients (peach-to-lilac, amber-to-navy) carry the visual weight; the chrome around them stays minimal.

## Layout
Fixed top nav: monospace logo left, center link cluster (Features dropdown, Teams, Pricing, Blog, Careers), Log in + pill "Start building" right — identical on dark and light pages. Feature-page heroes are compact: centered (or left-aligned) H1, one-line subhead, single orange CTA, then a full-width 16:9 rounded media panel (photo or 3D render). Body rhythm is a strict zig-zag: alternating text-block/screenshot two-column rows, each row repeating the orange CTA. Home page is a long scroll of full-bleed themed bands — dark, solid orange, white, dark again. Every page closes the same way: 3-column testimonial card, giant "Idea to software, fast" pixel-glitch headline, orange CTA, dense multi-column footer. Whitespace is generous; content sits in a wide centered container (~1200px).

## Components
Buttons: small-radius (4-6px) rectangles; primary is flat orange fill with white 13px label, no shadow, no border; secondary is light gray fill or ghost. CTAs are deliberately small relative to the layout. Cards: large-radius (16-24px) panels — testimonial band in soft gray (light pages) or desaturated slate (dark pages), feature media on pastel gradient backgrounds, frosted-glass feature tiles on the deployments page (blurred backdrop, thin white border). Pricing: bordered white plan columns, highlighted plan gets a pale orange tint + orange CTA; long comparison table with thin hairline rows and check marks. Inputs barely appear; the toggle (Monthly/Yearly) is a small segmented pill. Editor mockups use the dark IDE theme with orange/blue collaborator cursor name-tags.

## Signature
- Pixel-glitch letterforms in headline taglines ("Idea to software, fast") — letters partially decomposed into bitmap squares.
- Single orange (#E4592B) reserved almost exclusively for CTAs and the four-dot logo mark; repeated "Start building" at every scroll stop.
- Contrast of tiny flat buttons against oversized, heavily rounded media panels with pastel multi-color gradients.
- Alternating full-bleed theme bands on the home page: dark navy → solid orange → white, each self-contained.

## Do / Don't
**Do**
- Use one saturated orange for every primary CTA and keep it small, flat, and label-only ("Start building" repeated per section).
- Round media/screenshot containers heavily (16-24px) and set them on soft pastel or photographic gradient fills.
- Alternate dark (#0E1525) and white page themes while keeping nav, footer, and CTA styling identical across both.
- Close every page with the testimonial trio + giant glitch-type tagline + CTA sequence.
- Set the logo/wordmark and code snippets in monospace; keep marketing copy in the grotesque.

**Don't**
- Don't add gradients, shadows, or large sizes to buttons — CTAs here are flat, compact rectangles.
- Don't use orange for links, icons, or decoration; on-page accents are blue, amber, and lilac instead.
- Don't apply the pixel-glitch treatment to body copy or every heading — it's reserved for the recurring tagline.
- Don't crowd sections: each zig-zag row holds one short heading, 2-3 lines of copy, one CTA, one visual.
- Don't brighten the dark canvas toward pure black or neutral gray — it's a blue-tinted near-black.
