## Overview
A big-type, quasi-brutalist tech landing: enormous black condensed headlines on a stark white canvas, punctuated by a single vivid azure blue for CTAs and highlighted words. Isometric 3D "chain-link" illustrations in candy green/orange/blue/pink and halftone-dot textures give it a diagrammatic, technical-poster energy. Monospace type for microcopy and code panels reinforces the developer-protocol positioning.

## Layout
Slim top nav: wordmark left, uppercase text links center, solid blue CTA pill-adjacent button right. Hero is left-aligned mega-headline (two lines, one phrase colored blue) over a small eyebrow label, with a two-column follow-up (giant blue chevron glyph + paragraph) and an isometric illustration on the right. A full-width scrolling ticker/marquee banner announces news between sections. Sections alternate white and pale ice-blue (#D9ECF7) backgrounds; content alternates left-text/right-diagram and right-text/left-diagram in a steady zig-zag. Centered section headings ("How Axelar Works", "Use cases") in massive condensed black type. Generous vertical whitespace; roughly 12-col grid with wide margins.

## Components
- Buttons: solid azure blue fill, white uppercase small-caps monospace-feel label, small corner radius, no shadow; appear in nav, hero, and section CTAs. Text links in blue uppercase, sometimes as bare labeled links with no underline.
- Cards: flat, borderless blocks on tinted backgrounds; code-sample cards use dashed 1px black borders, square corners, pale blue fill, monospace code inside, with a blue "SEE DOCS" button below.
- Backers/Partners panels: pale blue rectangles with grayscale logo grids, square corners, no shadow.
- Inputs: newsletter field in footer is a plain rectangular input paired with a solid blue SUBMIT button.
- Footer: black wordmark, monospace tagline, multi-column uppercase link lists.

## Signature
- Ultra-heavy condensed black headlines with a single word or phrase flipped to azure blue.
- Isometric parallelogram "chain link" illustrations in green/orange/blue/pink with halftone-dot shadows and crosshair/axis marks, like technical figures (labeled "FIG. A").
- Dashed-border code panels and monospace microcopy giving an engineering-spec feel.
- Scrolling marquee news ticker as a full-width section divider.
- Strict two-tone rhythm: white sections alternating with pale ice-blue ones.

## Do / Don't
**Do**
- Set headlines huge, black-weight, condensed, tightly tracked; color one key phrase in the brand blue.
- Reserve the azure blue exclusively for CTAs, links, and headline highlights; keep everything else black-on-white.
- Use isometric flat-3D diagram illustrations with halftone dot textures instead of photos.
- Alternate white and pale ice-blue full-width section bands for rhythm.
- Use monospace uppercase for eyebrows, button labels, code, and footer taglines.

**Don't**
- Don't add drop shadows, gradient buttons, or large border radii — surfaces are flat and near-square.
- Don't use photography or realistic 3D renders; the visual language is diagrammatic vector art.
- Don't introduce extra accent colors into UI chrome — green/orange/pink live only inside illustrations.
- Don't center-align body copy in feature sections; text blocks are left-aligned in a zig-zag with diagrams.
- Don't soften the type: light or regular-weight display headings would break the poster-like impact.
