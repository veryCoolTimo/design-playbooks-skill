## Overview
Clean, product-forward SaaS minimalism: near-white canvas, deep navy headlines, gray body text, one bright blue used almost exclusively for CTAs and badges. The real color comes from large, crisp product screenshots and soft blue/violet gradient panels that frame them. Everything else stays quiet — thin borders, generous whitespace, small type.

## Layout
Slim top nav: logo left, three text links, single blue pill CTA right, on the page background with no bar fill. Hero is centered text ("Customer Relationship / in good hands" — navy first line, gray second) sitting on a soft blue gradient band, with a large floating product screenshot overlapping below it. Section rhythm on home: logo cloud → alternating one-word verb headings with a period ("Prospect.", "Close.", "Retain.", "Recycle.") each paired with a big UI screenshot or split two-column screenshot pair → bento-style feature card grid → testimonial cards → dark navy footer with newsletter input. Changelog is a single centered column timeline: date in the left gutter, entry title + "New"/"Launch Week" pill badge, screenshot card (often on a colorful gradient backdrop), short paragraph, "Read more" link. Content column is narrow (~640-900px) with wide empty margins.

## Components
Buttons: solid bright-blue pill, white text, small size, no visible shadow; secondary actions are plain text links. Badges: tiny pill chips ("New", status labels) in blue or tinted fills. Cards: white fill, 1px light-gray border, 8-14px radius, minimal or no shadow; screenshot frames get the same radius plus subtle drop shadow when floating over gradients. Kanban-style deal cards inside screenshots carry colored status dots (blue/orange/green/red) and dollar values. Inputs: footer newsletter field is a small dark rounded input with an adjacent pill button. Footer is a rounded dark-navy panel (#0E1420-ish) inset from the page edges.

## Signature
- One-word verb section headings ending in a period ("Prospect.", "Close.", "Retain.", "Recycle.") in bold navy, centered.
- Soft blue/violet gradient panels used as backdrops for hero and screenshot showcases against an otherwise near-white page.
- Product screenshots as the primary visual content — real CRM UI (kanban pipelines, tables) doing the storytelling.
- Rounded dark-navy footer card inset from the page, contrasting the light page above.
- Changelog timeline with left-gutter dates and gradient-framed screenshot cards per entry.

## Do / Don't
**Do**
- Keep blue (#2563EB) reserved for CTAs and small badges; let screenshots and gradients carry the rest of the color.
- Use large, legible product-UI screenshots with medium-rounded corners as section centerpieces.
- Write terse, centered section headings (1-3 words) with a gray one-line subhead beneath.
- Frame showcase imagery on soft blue-to-lavender gradient panels with rounded corners.
- End the page with an inset, rounded dark-navy footer containing the newsletter signup.

**Don't**
- Don't add heavy shadows, thick borders, or filled nav bars — chrome stays 1px light gray on white.
- Don't use illustration or stock photography; the product UI is the only imagery.
- Don't introduce more than one saturated brand color in the UI chrome — extra hues appear only inside screenshots and status dots.
- Don't use square-cornered buttons; CTAs are always small blue pills.
- Don't crowd sections — each screenshot block gets a full band of whitespace above and below.
