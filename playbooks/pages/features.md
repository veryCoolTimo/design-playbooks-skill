# Features Page Playbook
> Coverage: 19 examples from corpus (analyzed 19)

## Anatomy

Ordered skeleton observed across the corpus (n=19):

1. **Nav** — standard marketing nav; ~30% highlight the active "Features" item (canopy-is pill, goodnotes underline) or show a features sub-nav with anchor tabs (privy: "Auth for everyone / Embedded wallets / Powerful connectors / User management").
2. **Hero** — 100%. Headline + 1–2 line subhead + primary CTA. ~70% centered (buzzsprout, flox, framer, june, resend, sketch, planpoint), ~30% split text-left / product-visual-right (getcontrast, privy, kit, goodnotes). ~40% put an eyebrow pill/label above the headline naming the feature area ("Features" at flox, "VISITOR AUTHENTICATION" at gitbook, "Design Features" at lottielab, "# Supertags" treatment at tana).
3. **Hero product visual** — ~90%. Real product screenshot directly under/beside the headline (framer, june, kit, resend, synthesia, tana); illustration only at todoist.
4. **Social proof strip** — ~25% place a customer logo wall immediately after the hero (gitbook, june ×2, synthesia); others defer it to the bottom (canopy-is "Trusted by 30,000+ managers") or use quote cards early (tana).
5. **Feature body** — the core, three dominant patterns:
   - **Alternating zigzag rows** (image/text swapping sides), ~45%: buzzsprout, canopy-is, goodnotes, june, kit, lottielab, privy, todoist.
   - **Bento/card grids**, ~45%: flox (2 large + 3 small), framer (overview bento then deep dives), getcontrast (mosaic of screenshot + text cards), krepling, resend (use-case text grid), studio-design, synthesia (tabbed card grid), planpoint (catalog of ~40 small cards).
   - **Color-blocked full-bleed chapters** that change background per feature theme, ~26%: buzzsprout (navy/yellow/black bands), goodnotes (navy + lavender AI sections), getcontrast (pink/blue/cream), krepling, studio-design (alternating dark/light chapters).
   Most pages mix two of these (framer: bento overview → long-form single-column deep dives; krepling: chapter blocks containing card grids).
6. **Interleaved trust elements** — quotes between feature sections at canopy-is (after every feature), todoist (one-liner after every section), kit (large photo + quote video testimonials).
7. **Mid-page CTA repeats** — buzzsprout ("Start a podcast. Free for 90 days!" band a third of the way down), krepling (black CTA bar after every chapter), goodnotes ("Download on iOS" after each color block).
8. **Late-page social proof / results** — testimonial walls (june tweet wall, resend 4-up, todoist tweets + app-store review columns), case-study ROI cards (synthesia), investor logos (privy "Backed by").
9. **Final CTA band or card** — ~95%. Full-bleed color band (lottielab purple gradient, todoist red, kit blue) or contained gradient card (gitbook, getcontrast). Usually dual CTA: primary + secondary (flox "Download / Contact us", synthesia "Create an account / Create a free AI video").
10. **FAQ** — rare (~5%): only goodnotes has an accordion before the footer.
11. **Deep footer** — 100%; feature pages consistently link sibling feature pages in the footer (gitbook "Features" column, tana "Features of Tana" column).

## Patterns that work

- **Open with the product, not an abstraction.** Nearly every strong page puts a real UI screenshot within the first viewport: framer's editor under an orange glow, june's dashboard, resend's email console fading into black, synthesia's video editor with a play button. Tana frames screenshots in colored window chrome (purple/pink frames) to keep long screenshot runs visually distinct.
- **Benefit headline + capability subsections.** Heroes state an outcome ("Design sites faster." framer; "From overwhelmed to on top of it" todoist; "Make features people want" june), then each section names a concrete capability ("Breakpoints", "Figma Copy Paste" at framer; "Edit and annotate PDFs with ease" at goodnotes).
- **Interleave testimonials as section separators.** canopy-is places a quote + avatar between every single feature block; todoist drops a one-line customer quote after each zigzag row; kit alternates big photo-quote testimonials with feature sections. This paces a long page and re-validates each claim.
- **Bento overview → deep dives.** framer shows a compact bento grid summarizing all features, then expands each cell into a full section below; krepling opens with 3 index cards (Storefront / Workflows / Dashboard) that mirror the page's three chapters — the grid doubles as a table of contents.
- **Color-block per feature theme.** buzzsprout switches background color per feature family (navy for Magic Mastering, yellow for mobile, black for the player); goodnotes uses navy for AI features and lavender for study features; studio-design alternates dark/light chapters. The background shift signals "new topic" without headers doing all the work.
- **Bulleted micro-copy beats paragraphs.** lottielab replaces body paragraphs with bullet lists whose lead-ins are bolded ("Vector files:", "Transform:"); sketch uses checkmark lists ("Dark and light user interfaces", "Toolbar support") inside its "Made for macOS" block; goodnotes uses feature-chip pills ("Highlight", "Shape Tool", "Lasso Tool").
- **"New" / "Coming soon" badges create momentum.** planpoint tags cards with "New" and "Coming up"; gitbook badges "Dynamic documentation — Coming soon"; synthesia marks "Gestures New!" and "Voice cloning New!"; sketch has a "Recently updated" carousel. Signals an actively developed product.
- **Each section links out, never dead-ends.** canopy-is ends every block with "Learn more about X →"; planpoint puts "View this feature" on all ~40 cards; framer attaches a "Watch lesson" pill to every capability; privy's "Explore our powerful features" row links to sibling feature pages.
- **Quantified friction-killers near the CTA.** "Set up Privy in 9 minutes" with actual code, "Set up June in 2 minutes", buzzsprout "Free for 90 days", resend "Start now $0/mo", getcontrast "Everything webinars in 2 clicks". Concrete numbers, placed where the reader decides.
- **Tabs or a catalog when features are numerous.** synthesia tabs its grid ("AI Avatars & Voices / Media & Integrations / Sharing & Exports / Security"); planpoint goes full catalog with a uniform icon-card grid. Works only with consistent card anatomy: icon, title, 2-line text, link, optional badge.

## Variants by style

- **Light minimal / SaaS-clean** (buzzsprout, flox, lottielab, privy, synthesia, todoist, goodnotes, planpoint): white base, generous whitespace, one accent color; zigzag rows or icon-card grids; screenshots in soft-shadow cards or rounded panels. Flox and privy add a single soft radial/gradient wash behind the hero only. Pastel section bands (goodnotes lavender, buzzsprout yellow) inject variety without breaking the light feel.
- **Dark tech** (framer, gitbook, resend, canopy-is): near-black base, glow/aurora gradients as hero backdrop (framer orange, gitbook teal-violet aurora, resend faint green spotlight), screenshots blend into the dark background instead of sitting in cards. Typography does emphasis work: resend pairs a serif display hero with mono eyebrows; gitbook uses monospace eyebrow tags. CTAs are the brightest element on the page (gitbook yellow, canopy-is purple).
- **Colorful / playful** (getcontrast, krepling, studio-design as hybrid, todoist's illustrations): full-bleed pastel or gradient section blocks, mosaics mixing dark and light cards in one grid (getcontrast), hand-drawn accents and character illustrations (todoist), gradient chapter labels and integration pills in brand colors (krepling). Playful pages still keep body copy small, neutral, and highly structured.
- **Editorial / typographic** (kit, resend's serif hero, tana): cream or tinted backgrounds, oversized display type ("Moving to Kit couldn't be easier"), photographic customer portraits at large size (kit), narrative section flow reading like an essay ("As little as a label… or as big as an app." at tana) rather than a spec sheet.
- **Hybrid dark/light chaptering** (studio-design, sketch's "Made for macOS", june's navy feature scroller): a mostly-light page drops into one long dark chapter for its flagship feature set, then returns to light — the inversion earns disproportionate attention for that one section.

## Anti-patterns

- **Icon-only feature grids with no product evidence.** planpoint's 40-card catalog relies on tiny generic icons; without screenshots, capabilities feel asserted rather than shown. Every stronger page (framer, tana, goodnotes) proves each claim with a real UI crop.
- **No table of contents on very long pages.** buzzsprout and framer run 15+ sections with no anchor nav; only studio-design (side anchor list) and privy/synthesia (sub-nav tabs) help readers jump. If the page exceeds ~8 sections, corpus leaders add navigation.
- **Uniform section rhythm with zero variation.** Pages that alternate the same two-column row forever without color shifts, quotes, or CTAs between (lottielab comes closest) read as monotonous; the best pages break rhythm every 2–3 sections with a band, quote, or grid.
- **CTA only at top and bottom.** Corpus leaders repeat conversion points mid-page (krepling after every chapter, goodnotes per block, buzzsprout a third of the way down). Pages 10,000+ px tall with two CTAs waste scroll-depth intent.
- **Stock imagery and abstract illustration in place of UI.** Conspicuously absent across all 19: no stock photos of "people at laptops." Even illustration-heavy todoist shows real app screenshots beside every illustrated accent. Photography appears only as customer portraits attached to quotes (kit, canopy-is, synthesia).
- **Unbadged staleness.** None of the corpus dates features, but several actively signal freshness ("New", "Coming soon", "Recently updated"). Omitting these on a feature catalog makes the product look frozen.
- **Wall-of-paragraph body copy.** Corpus body text is capped at ~2–3 short lines per feature; anything longer is converted to bold-lead bullets (lottielab), checkmarks (sketch), or chips (goodnotes). No page uses multi-paragraph feature descriptions.
- **Weak or missing final close.** tana ends on resource/article cards without a CTA band — the one page in the corpus that lets exit intent fizzle; the other 18 end on a high-contrast CTA.

## Checklist

1. Hero: outcome-focused headline + subhead + primary CTA, with a real product screenshot in the first viewport; add an eyebrow pill naming the feature area if this is one of several feature pages.
2. Decide the body pattern by feature count: ≤6 features → alternating zigzag rows; 7–15 → bento/overview grid with deep-dive sections; 15+ → tabbed grid or uniform card catalog (with screenshots, not just icons).
3. Prove every capability with a cropped, framed UI screenshot or short UI mock — never an abstract icon alone, never stock photography.
4. Keep feature copy to one heading + ≤3 lines, or bold-lead bullets/checkmark lists; give each section an outbound link ("Learn more →" / "Watch lesson").
5. Break rhythm every 2–3 sections: change background color for a new feature theme, insert a one-line customer quote with avatar, or drop a mid-page CTA band.
6. Add social proof twice: a logo wall or quote cards near the top, and a testimonial wall / case-study numbers near the bottom.
7. Badge recency: "New", "Coming soon", or a "Recently updated" module.
8. Close with a full-width high-contrast CTA band or gradient card, dual buttons (primary action + low-commitment secondary), plus one friction-killer stat ("set up in X minutes", "free for X days", "$0/mo").
9. On pages >8 sections, add navigation: sticky sub-nav tabs, side anchor list, or an overview grid whose cells anchor-link to sections.
10. Footer: include a features column linking sibling feature pages.

## Best references

- **framer** — dark-tech benchmark: bento overview expanding into per-capability deep dives, each with screenshot + "Watch lesson" link.
- **canopy-is** — quote-between-every-feature pacing; cleanest zigzag rhythm on a dark page with consistent "Learn more →" exits.
- **goodnotes** — color-blocked theming (navy AI, lavender study) with 3-card grids and a repeated platform-specific CTA per block.
- **gitbook** — single-feature deep page done right: aurora dark styling, integration card grid, "coming soon" roadmap badge, gradient CTA card.
- **getcontrast** — playful mosaic grids mixing dark/light cards inside pastel full-bleed chapters; strong "2 clicks" friction-killer hero.
- **resend** — typographic dark minimalism: serif hero, mono eyebrows, use-case text grid, screenshots dissolving into background.
- **todoist** — light/illustrated variant with a quote after every section and the best end-of-page social proof stack (tweets + store reviews).
- **krepling** — chaptered architecture: index cards mirroring page sections, "Additional features" sub-lists, CTA bar repeated after every chapter.
