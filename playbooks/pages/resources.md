# Resources Page Playbook
> Coverage: 36 examples from corpus (analyzed 20)

## Anatomy

"Resources" is an umbrella: the corpus splits into blog/article indexes (notion, hydra, front, lattice, metaview), template/example galleries (cakeequity, equals, artboard-studio, eraser, fable), case-study hubs (brainfishai, jasper, mintlify), webinar/course libraries (canopy-is, clay-com), guide directories (oysterhr), report libraries (default), and help articles (dub). The skeleton below holds across all of them:

1. **Global nav** (100%) — sometimes plus a page-local sub-nav: front runs a "Front Page" magazine bar with its own categories; dub and clay-com add a persistent left sidebar.
2. **Hero title block** (100%). Variants: centered title + one-line subtitle (~50%: canopy-is, lattice, jasper, oysterhr, equals); hero with embedded utility — search bar (clay-com, oysterhr), email capture (front, default), or video (brainfishai); oversized typographic statement (~5%: eclipse's viewport-wide "Resources"). ~30% add an eyebrow label naming the content type ("RESOURCE HUB" on cakeequity, "Case studies" on brainfishai, "CLAY UNIVERSITY" on clay-com).
3. **Featured/priority slot** (~45%) — one item promoted above the grid: latest post (kindsight, metaview, hydra), upcoming live event (canopy-is), flagship report (default), hero case study with stat bar (jasper), "Get started today" course (clay-com).
4. **Filtering/browsing controls** (~65%): horizontal category pills or tabs (artboard-studio, front, hydra, default, metaview); left filter rail with search + checkbox/chip facets for big libraries (eraser, lattice, notion, clay-com); search + "Filter by" dropdown (kindsight, oysterhr).
5. **Card grid** (95%). 3-column is the default (~45%: lattice, kindsight, canopy-is at 4, default, metaview); 2-column for richer cards (~30%: brainfishai, notion, hydra, clay-com, equals, mintlify); masonry for visual template libraries (artboard-studio, fable).
6. **Pagination**: "Load more" button (lattice, kindsight, eraser, artboard-studio) or numbered pages (notion, oysterhr). ~25% render everything and get very long (metaview, mintlify, fable).
7. **Newsletter capture** (~50%): band before footer (lattice, hydra), in the hero (front, default, metaview toolbar), or as a card embedded inside the grid itself (kindsight, brainfishai footer).
8. **Product CTA band** (~70%) — full-width conversion section restating product value before the footer (cakeequity, jasper, clay-com, equals, artboard-studio, brainfishai, oysterhr, hydra, mintlify).
9. **Fat footer** (100%), usually with a "Resources" column that deep-links sibling resource types.

Card anatomy (near-universal): thumbnail → type/category tag → title → 1–3 line excerpt → meta (author avatar, date, read time) → link or whole-card click.

## Patterns that work

- **Metric-led titles for case studies.** brainfishai titles every card with the outcome ("How Madpaws saw 636% ROI within 5 months"); jasper attaches a color-coded stat chip to every logo card ("3,000+ hours saved", "166% increase in organic traffic"); default bakes sample sizes into report titles ("Analysis of 421 Successful Launches"). The number does the selling before anyone clicks.
- **A systematic branded thumbnail treatment beats per-post improvisation.** metaview renders every post on the same brand-gradient wave; default gives all reports one purple cover template; canopy-is commissions a consistent pastel-illustration set; hydra mixes line-art elephants in one palette. The grid reads as one product, and posts without natural imagery still look intentional.
- **Featured slot before the grid creates hierarchy.** kindsight opens with a full-width featured post on a tinted panel; canopy-is pins an "Upcoming live event" card with date/duration/speaker; hydra leads with an orange featured recap card twice the height of the rest; default puts the flagship report in the hero.
- **Faceted filter rail for 30+ item libraries.** eraser stacks search + Example Type + Use Case + Eraser Features + Technologies as chip groups in a left rail; lattice uses topic checkboxes + search; clay-com's sidebar splits Learn (Courses/Guides/Claybooks) from Community. Pills alone (front, hydra) are enough only for a handful of categories.
- **Action-verb links on functional resources.** cakeequity's template cards each show a mini document preview and end in "Use this template →"; equals pairs a colored icon with a one-line job-to-be-done per template; artboard-studio labels every mockup thumbnail by type. Not "Read more" — say what the click does.
- **Cross-sell sibling resources from the hub.** clay-com ends with a "Learn with our resources" row linking University, blog, integrations, and templates; front curates three text-link columns (Front News / Popular / Guest Writers) below the grid; eclipse's hero anchor bar (Docs / Solana VM / FAQs) jumps between its resource blocks.
- **Author identity on editorial content.** notion, hydra, and metaview show avatar + name + role on every card; notion's hand-drawn portraits double as thumbnails. Case-study pages swap author for customer logo (jasper, mintlify, brainfishai).
- **Type + effort labels make mixed grids scannable.** kindsight tags every card (Post / Case Study / On-demand Webinar) and adds read time ("21 mins read", "Watch now"); hydra chips each post (Launch Week / Engineering / Partnership) with a date; lattice color-tags topic categories.
- **Newsletter capture placed where the reader already is.** kindsight embeds a "Be the first to read our resources" card inside the grid at position ~6; front puts the subscribe field directly in the hero; lattice closes with a one-field "One email. Every week." band.

## Variants by style

- **Light minimal** (lattice, notion, metaview, default, front, artboard-studio): white/near-white ground, cards defined by thumbnail + hairline borders or bare whitespace, small colored category tags as the only accent. Restraint pushes all differentiation into the thumbnail system — lattice's varied art vs metaview's repeating waves shows how much that choice matters.
- **Dark tech** (eraser, canopy-is, hydra, fable, mintlify, eclipse): near-black ground with slightly lighter card surfaces; thumbnails and accent colors (hydra's orange, eraser's diagram colors) carry all vibrancy. Filter rails and mono/technical type fit here (eraser). Eclipse pushes to brutalist: flat color blocks, giant lowercase wordmark footer, accordion FAQs instead of cards.
- **Colorful/playful** (brainfishai, clay-com, equals): illustrated brand world extends into the cards themselves — brainfishai gives each case study a wavy underwater header in a rotating palette; clay-com wraps courses in doodle-covered art and lands on a 3D garden CTA; equals uses flat color icon tiles and a pixel-stripe divider. Cards stay legible because text sits on white/cream below the colorful header.
- **Editorial serif** (jasper, kindsight, front): serif display headlines ("Resources Hub", "Jasper helps 100,000+ customers"), warm neutral grounds, magazine conventions — read times, guest-writer columns, featured story layouts. Jasper pairs the serif voice with hard data chips so it never reads as fluffy.
- **Docs utility** (dub): help-center article pattern — breadcrumbs, right-rail table of contents, annotated screenshots, tip callouts, related-articles list, emoji feedback widget. No marketing sections at all; don't mix this template with hub templates.

## Anti-patterns

- **Undifferentiated infinite grid.** metaview renders 60+ posts with near-identical wave thumbnails and no load-more cap — by row 10 nothing is distinguishable; mintlify's customer list becomes a wall of tiny same-shaped logo cards. Cap the initial render (~9–15 items) and vary the featured slot, as lattice and kindsight do.
- **Interchangeable stock photography.** kindsight's grid of "person at laptop" photos means titles must do 100% of the scanning work — the weakest part of an otherwise strong page. Every other strong example uses illustration, product shots, or a generated cover system instead.
- **"Read more" as the only link language on functional resources.** The template galleries that convert (cakeequity, equals) never do this — they use "Use this template", "Start learning", "Watch now".
- **No content-type signal in the hero.** The corpus almost always names the format immediately (Webinars, Templates, Customer Stories, Reports); a bare "Resources" heading with mixed unlabeled cards forces the user to infer what each click costs. Eclipse gets away with the bare word only because its anchor bar immediately enumerates the sections.
- **Filters absent on large mixed libraries.** Conspicuously avoided: every corpus example past ~12 items adds pills, tabs, a rail, or search. fable's 60-item masonry with only sidebar category links is the weakest browsing experience in the set.
- **Newsletter only in the footer.** Half the corpus surfaces capture in the hero, grid, or a dedicated band; burying it below the fat footer (where brainfishai's lives) makes it nearly invisible on a page this long.
- **Skipping the product CTA.** Resources pages are top-of-funnel; the corpus rarely lets the page end without a conversion band (canopy-is and notion are the exceptions, and both at least push app download / product links in the footer).

## Checklist

1. Pick the subtype first (blog index / templates / case studies / webinars / multi-type hub) — it determines card anatomy and link verbs.
2. Hero: eyebrow naming the content type + H1 + one-line value subtitle; embed search or email capture here if the library is large or editorial.
3. Add one featured slot above the grid: latest post, upcoming event, or flagship asset — visually distinct (tinted panel, larger card, or stat bar).
4. Design a repeatable thumbnail system (brand gradients, illustration set, or templated covers) so every card looks owned even without unique art.
5. Card = thumbnail + type tag + benefit-or-metric title + short excerpt + meta (author/date/read time or stat chip) + action-verb link.
6. Grid: 3-col for compact cards, 2-col for rich ones; show 9–15 items, then "Load more" or pagination — never dump the full archive.
7. Filtering: category pills for ≤6 categories; search + faceted left rail for larger or multi-dimension libraries.
8. Place newsletter capture in the flow (hero, in-grid card, or pre-footer band), not only in the footer.
9. Close with a full-width product CTA band restating the core value prop with primary + secondary buttons.
10. Footer: include a Resources column cross-linking sibling resource types (blog, webinars, templates, help center).

## Best references

- **clay-com** — the richest hub: sidebar IA, featured course, carousels, and a cross-sell row tying University/blog/templates together.
- **kindsight** — cleanest editorial hub anatomy: featured post, type tags + read times, in-grid newsletter card, load more.
- **eraser** — model faceted filter rail (search + type + use case + technology chips) for a large example gallery on dark.
- **brainfishai** — case-study cards done right: metric-led titles, quote excerpts, systematic colorful headers per customer.
- **jasper** — customer-story wall where every card carries a color-coded outcome stat chip; serif editorial voice + hard data.
- **hydra** — dark blog index with category chips, strong featured card, consistent illustration system, subscribe band.
- **cakeequity** — template gallery pattern: document mini-previews, "Use this template" verbs, supporting proof video, loud CTA band.
- **front** — magazine treatment: page-local nav, hero subscribe, featured story, and curated link columns below the grid.
